# SITWallet SQL Schema

The executable schema is in `sql/001_schema.sql`. It creates the `sitwallet` database, tables, seed data, and stored procedures.

## Tables

### roles

RBAC role lookup.

| Column | Type | Notes |
| --- | --- | --- |
| role_name | VARCHAR(30) PK | `customer`, `support`, `admin` |
| description | VARCHAR(255) | Human-readable role description |
| created_at | TIMESTAMP | Creation time |

### users

Holds registration, login, MFA, and RBAC state.

| Column | Type | Notes |
| --- | --- | --- |
| user_id | CHAR(36) PK | UUID generated by backend |
| full_name | VARCHAR(120) | User legal/display name |
| email | VARCHAR(255) UNIQUE | Lower-cased email |
| phone_number | VARCHAR(32) UNIQUE NULL | Captured during MFA setup |
| password_hash | VARCHAR(255) NULL | bcrypt hash after final registration |
| totp_secret_ciphertext | TEXT NULL | AES-256-GCM encrypted TOTP secret |
| totp_secret_iv | VARCHAR(32) NULL | TOTP encryption IV |
| totp_secret_tag | VARCHAR(32) NULL | TOTP auth tag |
| mfa_enabled | BOOLEAN | Must be true for active users |
| primary_currency | CHAR(3) NULL | FK-like value from currencies |
| role_name | VARCHAR(30) FK | RBAC role |
| status | ENUM | `registration_started`, `mfa_pending`, `password_pending`, `active`, `suspended`, `closed` |
| failed_login_count | INT | Used for temporary lockout |
| locked_until | DATETIME NULL | Lockout expiry |
| terms_accepted_at | DATETIME NULL | Set during final registration |
| last_login_at | DATETIME NULL | Updated after MFA login |
| created_at, updated_at | TIMESTAMP | Audit timestamps |

### currencies

Hardcoded currency catalog and USD reference values.

| Column | Type | Notes |
| --- | --- | --- |
| currency_code | CHAR(3) PK | ISO-style code |
| currency_name | VARCHAR(80) | Display name |
| symbol | VARCHAR(8) | Display symbol |
| decimal_places | TINYINT | Frontend formatting hint |
| rate_to_usd | DECIMAL(20,10) | Hardcoded reference rate |
| high_24h_to_usd | DECIMAL(20,10) | Hardcoded 24h high |
| low_24h_to_usd | DECIMAL(20,10) | Hardcoded 24h low |
| change_24h_pct | DECIMAL(10,4) | Hardcoded 24h percent change |
| is_active | BOOLEAN | Soft availability flag |
| updated_at | TIMESTAMP | Last update |

Seeded currencies: `USD`, `SGD`, `EUR`, `GBP`, `JPY`, `MYR`, `AUD`, `CAD`, `INR`.

### platform_settings

Admin-managed fees and limits.

| Column | Type | Notes |
| --- | --- | --- |
| setting_key | VARCHAR(80) PK | Example: `exchange_fee_percent` |
| setting_value | DECIMAL(20,8) | Numeric value |
| setting_unit | ENUM | `percent`, `amount`, `boolean` |
| description | VARCHAR(255) | Meaning |
| updated_by | CHAR(36) FK NULL | Admin user |
| updated_at | TIMESTAMP | Last update |

Seeded settings:

- `exchange_fee_percent`
- `transfer_fee_percent`
- `top_up_fee_percent`
- `daily_top_up_limit`

### exchange_rates

Complete ordered currency-pair table generated from the seeded currencies.

| Column | Type | Notes |
| --- | --- | --- |
| rate_id | BIGINT PK | Auto-increment |
| from_currency | CHAR(3) FK | Source currency |
| to_currency | CHAR(3) FK | Target currency |
| rate | DECIMAL(20,10) | Conversion multiplier |
| high_24h | DECIMAL(20,10) | Pair high |
| low_24h | DECIMAL(20,10) | Pair low |
| change_24h_pct | DECIMAL(10,4) | Pair change |
| source | VARCHAR(60) | `seed` or `admin` |
| updated_by | CHAR(36) FK NULL | Admin user |
| updated_at | TIMESTAMP | Last update |

Unique key: `(from_currency, to_currency)`.

### wallets

One wallet per user/currency.

| Column | Type | Notes |
| --- | --- | --- |
| wallet_id | CHAR(36) PK | UUID |
| user_id | CHAR(36) FK | Wallet owner |
| currency_code | CHAR(3) FK | Wallet currency |
| balance | DECIMAL(20,8) | Non-negative balance |
| is_active | BOOLEAN | Soft availability flag |
| created_at, updated_at | TIMESTAMP | Audit timestamps |

Unique key: `(user_id, currency_code)`.

### recipients

Saved recipient list.

| Column | Type | Notes |
| --- | --- | --- |
| recipient_id | CHAR(36) PK | UUID |
| owner_user_id | CHAR(36) FK | User who saved the recipient |
| recipient_user_id | CHAR(36) FK | Registered target user |
| nickname | VARCHAR(120) NULL | Optional display label |
| created_at | TIMESTAMP | Creation time |

Unique key: `(owner_user_id, recipient_user_id)`.

Immutable ledger for top-ups, transfers, and exchanges.

## Stored Procedures

### sp_create_wallet

Creates or reactivates a wallet for an active user and supported currency. Used by `POST /api/wallets`.

### sp_finalize_registration

Finalizes staged registration by setting the bcrypt password hash, primary currency, terms acceptance timestamp, active status, and primary wallet.

### sp_add_recipient

Adds a registered active user to another user's recipient list. Rejects self-recipient entries.

### sp_top_up

Performs top-up atomically:

- Validates active user and currency.
- Enforces `daily_top_up_limit`.
- Deducts `top_up_fee_percent`.
- Creates/reactivates wallet if needed.
- Credits net amount.
- Inserts immutable `TOP_UP` transaction.

### sp_exchange

Performs same-user currency exchange atomically:

- Validates rate and wallets.
- Deducts `exchange_fee_percent` from source amount.
- Debits source wallet.
- Creates/reactivates target wallet if needed.
- Credits converted amount.
- Inserts immutable `EXCHANGE` transaction.

### sp_transfer

Performs transfer atomically:

- Validates active sender and recipient.
- Gets rate for cross-currency transfers.
- Deducts `transfer_fee_percent` from send amount.
- Debits sender wallet.
- Creates/reactivates recipient wallet if needed.
- Credits converted amount.
- Inserts immutable `TRANSFER` transaction.

### sp_get_dashboard

Returns multiple result sets for dashboard:

1. Total estimated balance in primary currency.
2. Wallet balances.
3. Recent transactions.
4. Exchange rates.

### sp_update_platform_setting

Admin-only stored procedure to update fee and limit settings.

### sp_log_audit

Writes audit rows from backend services.

## RBAC

`customer`:

- Can register, login, manage own wallets, top up, exchange, transfer, and view own history.

`support`:

- Can list users and audit logs.
- Cannot move funds or update settings.

`admin`:

- Can update exchange rates, platform settings, user status, and roles.
- Can access support-level read endpoints.

## Security Notes

- Passwords are stored only as bcrypt hashes.
- TOTP secrets are encrypted using AES-256-GCM with `APP_ENCRYPTION_KEY`.
- Refresh tokens are stored as SHA-256 hashes and rotated on refresh.
- Unsafe HTTP methods require CSRF token and cookie match.
- Financial writes are done in stored procedures with transactions and row locks.
- Wallet balances have database-level non-negative checks.
- Foreign keys preserve ledger integrity.
